# Схема базы данных (Cloudflare D1)

> База: `olesya-messenger-db` | database_id: `850a02fd-68ce-40be-8cfb-63ce2325f3a1` | регион: WEUR
> Применение схемы: `wrangler d1 execute olesya-messenger-db --file=schema.sql`
> Полный SQL — в файле `03-cloudflare-backend/schema.sql`.

База содержит 5 рабочих таблиц. Ниже — назначение каждой.

---

## om_users — пользователи

Хранит зарегистрированных пользователей мессенджера.

| Поле | Тип | Описание |
|---|---|---|
| id | TEXT PRIMARY KEY | Уникальный ID пользователя |
| phone | TEXT UNIQUE | Номер телефона в формате E.164 (`+79001234567`) |
| name | TEXT | Отображаемое имя |
| avatar | TEXT | Эмодзи-аватар (по умолчанию 🙂) |
| color | TEXT | Цвет аватара (по умолчанию `#6C7BFF`) |
| pass_hash | TEXT | Хэш облачного пароля (пусто, если не задан) |
| pass_hint | TEXT | Подсказка к паролю |
| online | INTEGER | 1 если в сети, 0 если оффлайн |
| last_seen | INTEGER | Время последней активности (мс с эпохи) |
| created_at | INTEGER | Время регистрации |

---

## om_otp — одноразовые коды (резервный Custom OTP)

Хранит SMS-коды для режима Custom OTP (когда не используется Firebase). Сырой код нигде не хранится — только его HMAC-хэш.

| Поле | Тип | Описание |
|---|---|---|
| rowid | INTEGER PK AUTOINCREMENT | Внутренний ID записи |
| phone | TEXT | Номер, на который отправлен код |
| code_hash | TEXT | HMAC-SHA256 кода (ключ — SESSION_SECRET) |
| expires_at | INTEGER | Срок действия кода (мс) |
| attempts | INTEGER | Счётчик неверных попыток (блок после 5) |
| created_at | INTEGER | Время создания |

**Защита:** не более 3 кодов за 10 минут на номер; код живёт 10 минут; после 5 неверных попыток — блокировка.

---

## om_sessions — сессии (токены авторизации)

Хранит активные сессии после успешного входа.

| Поле | Тип | Описание |
|---|---|---|
| token | TEXT PRIMARY KEY | Session-токен (передаётся в заголовке `x-session-token`) |
| user_id | TEXT | ID пользователя (пусто, пока регистрация не завершена) |
| phone | TEXT | Номер телефона |
| expires_at | INTEGER | Срок действия (30 дней для завершённой регистрации) |
| created_at | INTEGER | Время создания |

**Логика:** при Firebase-входе или верификации OTP создаётся токен. Если пользователь новый — токен временный (30 минут) до завершения профиля; после регистрации — 30 дней.

---

## om_messages — сообщения

Хранит сообщения для синхронизации между устройствами.

| Поле | Тип | Описание |
|---|---|---|
| id | TEXT PRIMARY KEY | ID сообщения |
| chat_key | TEXT | Ключ чата (отсортированные ID участников через `__`) |
| from_id | TEXT | ID отправителя |
| to_id | TEXT | ID получателя |
| type | TEXT | Тип: text / photo / voice / vnote |
| text | TEXT | Текст сообщения |
| data_url | TEXT | Данные медиа (base64) |
| dur | INTEGER | Длительность (для голосовых/видео-кружков) |
| orig | TEXT | Оригинальный текст (до перевода) |
| orig_lang | TEXT | Язык оригинала |
| reactions | TEXT | Реакции (JSON) |
| read_by | TEXT | Кем прочитано (JSON-массив) |
| t | INTEGER | Время отправки |

---

## om_bus — шина событий (real-time через polling)

Очередь событий для реал-тайм обновлений. Клиент опрашивает каждые 2.5 секунды.

| Поле | Тип | Описание |
|---|---|---|
| seq | INTEGER PK AUTOINCREMENT | Порядковый номер (для пагинации опроса) |
| to_id | TEXT | Кому событие (`*` = всем) |
| payload | TEXT | Полезная нагрузка (JSON) |
| t | INTEGER | Время |

**Как работает:** клиент шлёт событие через `POST /bus`, получатели забирают через `GET /poll?uid=...&since=seq`. Типы событий: новое сообщение, входящий звонок, набор текста, presence (онлайн/оффлайн), вход с устройства.

## Текущее состояние базы

- Зарегистрировано пользователей: 1 (тестовый — Константин, `+70000000001`)
- Все 5 таблиц и индексы созданы
- База готова к приёму реальных пользователей после подключения Blaze

## Полезные SQL-запросы

```sql
-- Сколько пользователей
SELECT COUNT(*) FROM om_users;

-- Список пользователей
SELECT id, phone, name, created_at FROM om_users ORDER BY created_at DESC;

-- Активные сессии
SELECT token, user_id, phone, expires_at FROM om_sessions
  WHERE expires_at > strftime('%s','now')*1000;

-- Очистить просроченные OTP
DELETE FROM om_otp WHERE expires_at < strftime('%s','now')*1000;

-- Последние сообщения в чате
SELECT * FROM om_messages WHERE chat_key=? ORDER BY t DESC LIMIT 50;
```
