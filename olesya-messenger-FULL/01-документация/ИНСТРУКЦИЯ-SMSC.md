# Подключение SMSC.ru — российские SMS без VPN

После этих шагов пользователи с номерами +7 будут получать коды через SMSC.ru
(работает в России без VPN), а пользователи из других стран — через Firebase.

---

## Как это работает

Когда человек вводит номер:
- **+7 (Россия)** → код отправляется через SMSC.ru (ваш Cloudflare Worker)
- **любой другой** → код через Firebase

Пользователь этого не видит — для него просто «пришёл код».

---

## Шаг 1 — Регистрация на SMSC.ru

1. Откройте **https://smsc.ru**
2. Нажмите **«Регистрация»**
3. Заполните данные (нужен российский телефон и email)
4. Подтвердите аккаунт

---

## Шаг 2 — Пополнить баланс

1. В личном кабинете → раздел **«Финансы»** или **«Оплата»**
2. Положите для теста небольшую сумму (100–200 рублей — хватит на ~30–50 SMS)
3. Оплата: карта, ЮMoney, СБП и другие российские способы

---

## Шаг 3 — Узнать логин и пароль для API

API SMSC использует ваш **логин и пароль от личного кабинета** (те же, что для входа на сайт).

Рекомендуется создать отдельного подпользователя для API:
1. Личный кабинет → **«Настройки»** → **«Подпользователи»** (необязательно, но безопаснее)
2. Либо используйте основной логин/пароль

> ⚠️ Для отправки кодов подтверждения отдельная регистрация имени отправителя не нужна — SMSC отправит с общего номера. Текст «Kod dlya vhoda...» латиницей проходит модерацию автоматически.

---

## Шаг 4 — Добавить секреты в Cloudflare Worker

1. **dash.cloudflare.com** → Workers & Pages → **olesya-api** → **Settings** → **Variables and Secrets**
2. Нажмите **Add** и добавьте два секрета:

| Имя | Значение |
|---|---|
| `SMSC_LOGIN` | ваш логин на smsc.ru |
| `SMSC_PASSWORD` | ваш пароль на smsc.ru |

3. Нажмите **Deploy** (или код передеплоится сам)

---

## Шаг 5 — Обновить код воркера

Если вы ещё не залили обновлённый воркер с поддержкой SMSC:

1. **olesya-api** → **Edit code**
2. Ctrl+A → Delete → вставьте код из обновлённого **`worker-paste.js`** → **Deploy**

---

## Шаг 6 — Обновить index.html на GitHub

Залейте обновлённый `index.html` (с маршрутизацией РФ→SMSC) на GitHub:
- github.com/vs2145061-dotcom/olesya → Add file → Upload files → `index.html` → Commit

---

## Шаг 7 — Проверить

1. Откройте `https://olesya-api.vs2145061.workers.dev/health`
   - Должно быть `"sms":true` (значит SMSC секреты на месте)
2. Откройте приложение **без VPN**
3. Введите свой **реальный** российский номер (+7...)
4. Должна прийти настоящая SMS с кодом
5. Введите код → вход

---

## Стоимость

- Регистрация: бесплатно
- SMS: ~2-4 рубля за штуку
- Минимального платежа нет

При 1000 регистраций в месяц — примерно 2000-4000 рублей. Платите только за реально отправленные SMS.

---

## Если что-то не работает

**`"sms":false` в /health** — секреты SMSC_LOGIN / SMSC_PASSWORD не добавлены или воркер не передеплоен.

**SMS не приходит, в логах ошибка SMSC** — проверьте баланс на smsc.ru и правильность логина/пароля. Логи воркера: Cloudflare → olesya-api → Logs.

**«SMS-провайдер не настроен»** — воркер не видит секреты. Добавьте их и нажмите Deploy.

**Код приходит, но «неверный код»** — проверьте, что время на сервере SMSC и в приложении совпадает (обычно не проблема).

---

## Тестовые номера Firebase продолжают работать

Номера `+70000000001` / `123456` и `+70000000002` / `654321` (тестовые в Firebase) работают параллельно — удобно для отладки без трат на SMS. Реальные номера +7 идут через SMSC, тестовые — через Firebase.
