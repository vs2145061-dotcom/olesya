## ПОДГОТОВКА (один раз)

### Шаг 1 — Установить Node.js
Скачайте: **https://nodejs.org** → версия LTS → установите.
Проверка: откройте терминал (cmd/PowerShell на Windows) и введите `node --version` — должно показать v20 или v22.

### Шаг 2 — Создать аккаунт Expo
Зарегистрируйтесь на **https://expo.dev** (бесплатно, только email).

### Шаг 3 — Установить EAS CLI
```bash
npm install -g eas-cli
```

### Шаг 4 — Распаковать проект
Распакуйте `olesya-native-expo.zip` в любую папку.
Откройте терминал и перейдите в папку проекта:
```bash
cd путь/к/olesya-native
```

### Шаг 5 — Установить зависимости
```bash
npm install

# контакты телефона (импорт в телефонную книгу) — обязательно для нового импорта:
npx expo install expo-contacts
```

### Шаг 6 — Войти в Expo
```bash
eas login
```
Введите email и пароль от expo.dev.

### Шаг 7 — Привязать проект к Expo
```bash
eas init
```
Когда спросит «Would you like to create a new EAS project?» — нажмите Y.

## ANDROID

### Сборка APK (для установки напрямую — тест, друзья)

```bash
eas build -p android --profile preview
```

- Сборка занимает **5–15 минут** в облаке Expo (ваш компьютер можно закрыть)
- По завершении в терминале появится ссылка на скачивание APK
- Также ссылка придёт на email и будет на **expo.dev → Builds**

**Установка на телефон:**
1. Откройте ссылку на Android-телефоне
2. Скачайте APK
3. При установке разрешите «Установку из неизвестных источников»
4. Установите → иконка «Olesya Messenger» появится на рабочем столе

---

### Публикация в Google Play

#### Шаг 1 — Аккаунт разработчика Google Play
Зарегистрируйтесь: **https://play.google.com/console** (единоразово $25)
Заполните профиль разработчика.

#### Шаг 2 — Создать приложение в Play Console
- Нажмите «Создать приложение»
- Название: **Olesya Messenger**
- Язык: Русский
- Тип: Приложение
- Бесплатное/платное: Бесплатное

#### Шаг 3 — Собрать релизный Bundle
```bash
eas build -p android --profile production
```
Получите файл `.aab` (Android App Bundle) — это для Play Store.

#### Шаг 4 — Загрузить в Play Console
- Слева: **Тестирование → Внутреннее тестирование → Создать выпуск**
- Загрузите .aab файл
- Добавьте себя как тестировщика

#### Шаг 5 — Заполнить карточку приложения
Всё необходимое уже написано в папке `stores/android/`:
- `play-listing-ru.md` — описание на русском
- `play-listing-en.md` — описание на английском
- `data-safety.md` — раздел «Безопасность данных»

**Скриншоты:** сделайте 2–8 скриншотов прямо с установленного APK:
- Экран входа
- Список чатов
- Открытый чат
- Настройки

**Политика конфиденциальности:** опубликуйте файл `PRIVACY-POLICY.md` на GitHub Pages и вставьте ссылку.

#### Шаг 6 — Отправить на проверку
Перейдите в «Рабочая версия» → «Создать выпуск» → загрузите .aab → отправьте на проверку.
Ожидание: **1–7 дней**.

## iOS

### Требования
- **Apple Developer Program**: $99/год — **https://developer.apple.com/programs/**
- Mac не нужен — EAS собирает в облаке

#### Шаг 1 — Купить Apple Developer Program
Откройте ссылку выше → «Enroll» → выберите «Individual» → оплатите $99.

#### Шаг 2 — Сборка для iOS
```bash
eas build -p ios --profile production
```
При первом запуске EAS попросит Apple ID и пароль — введите. EAS автоматически создаст сертификаты и provisioning profiles.

Получите файл `.ipa`.

#### Шаг 3 — Создать приложение в App Store Connect
Откройте: **https://appstoreconnect.apple.com**
- «Мои приложения» → «+» → «Новое приложение»
- Платформы: iOS
- Название: **Olesya Messenger**
- Bundle ID: `com.olesya.messenger`
- SKU: `olesya-messenger-001`

#### Шаг 4 — TestFlight (тест перед публикацией)
```bash
eas submit -p ios --latest
```
В App Store Connect → TestFlight → добавьте себя → скачайте TestFlight на iPhone → установите приложение.

#### Шаг 5 — Заполнить карточку
Всё готово в `stores/ios/`:
- `appstore-listing-ru.md` — описание на русском
- `appstore-listing-en.md` — описание на английском
- `app-privacy.md` — раздел приватности

**Скриншоты:** нужны для iPhone 6.9" (iPhone 15 Pro Max) и iPhone 5.5":
- Размеры: 1290×2796 и 1242×2208 пикселей
- Сделайте скриншоты с симулятора (в eas.json уже есть профиль `preview` для симулятора)

#### Шаг 6 — Отправить на ревью
«Отправить на рассмотрение в App Store» → ожидание **1–3 дня**.

## ОБНОВЛЕНИЕ ПРИЛОЖЕНИЯ

Когда выходит новая версия:
1. Загрузите обновлённый `index.html` на GitHub Pages
2. Приложение автоматически подтянет обновление при следующем запуске (оно загружает UI с GitHub Pages)

Пересобирать и переотправлять в магазины **не нужно** — только если меняется нативный код (App.js, app.json, зависимости).

## ЧАСТЫЕ ВОПРОСЫ

**Ошибка «eas: command not found»**
→ Закройте терминал и откройте заново после установки EAS CLI.

**Сборка завершилась с ошибкой**
→ Откройте expo.dev → Builds → нажмите на сборку → посмотрите логи → напишите ошибку мне.

**Как проверить приложение до публикации?**
→ APK (Android): `eas build -p android --profile preview` → установите напрямую
→ iOS симулятор: `eas build -p ios --profile preview` → нужен Mac с Xcode

**Можно ли обновить иконку/название без пересборки?**
→ Нет, иконка и название зашиты в бинарный файл. При изменении нужна новая сборка.
