# Полная история ошибок Olesya Messenger

> Хронология ВСЕХ проблем за весь проект и точные решения каждой.
> Разбито по категориям. В конце — сводка главных уроков.
> Это справочник: связка Firebase + Cloudflare + WebView + SMSC даёт типовые ошибки,
> здесь они все собраны.

---

## Категория A: Развёртывание и хостинг

### A1. GitHub Pages выдаёт 404
**Симптом:** при открытии сайта — 404, файл не найден.
**Причина:** файл загрузили под именем `olesya-messenger.html`. GitHub Pages для корневого URL ищет строго `index.html`.
**Решение:** переименовать в `index.html` (нижний регистр) перед загрузкой.

### A2. Cloudflare — «загрузчик не поддерживает проекты со сборкой»
**Причина:** попытка загрузить `worker.js` через раздел **Pages** (для сайтов).
**Решение:** воркер создаётся в разделе **Workers** (не Pages), код вставляется в редактор Edit code → Deploy, а не загружается файлом.

## Категория B: Firebase Authentication

### B1. `auth/internal-error` при отправке кода
**Причина:** домен не в списке авторизованных Firebase.
**Решение:** Firebase Console → Authentication → Settings → Авторизованные домены → добавить `vs2145061-dotcom.github.io`.
**Урок:** в Firebase Phone Auth `auth/internal-error` чаще всего = неавторизованный домен, а не внутренний сбой.

### B2. Домен пустой `""` в ошибке
**Причина:** приложение открыто через `file://` (двойной клик по файлу), а не `https://`.
**Решение:** открывать только по адресу GitHub Pages. Firebase не работает с `file://`.

### B3. `auth/billing-not-enabled`
**Причина:** Firebase Phone Auth требует план **Blaze**, даже при бесплатных 10 000 SMS/мес.
**Решение (полное):** подключить Blaze, привязав карту. Из РФ карты Мир/Visa не принимаются Google — нужна виртуальная карта (Aifory Pro) или карта Казахстана/Турции, при привязке указывать страну карты.
**Решение (для разработки):** добавить тестовый номер в Firebase Console → работает без Blaze.

### B4. reCAPTCHA «has already been rendered in this element»
**Причина:** верификатор reCAPTCHA пересоздавался на том же DOM-элементе.
**Решение:** переиспользовать существующий (`if(window._rcVerifier)return _rcVerifier`), сбрасывать только при реальной ошибке. Не вызывать `render()` явно — Firebase делает это сам внутри `signInWithPhoneNumber()`.
**Урок:** reCAPTCHA-верификатор создаётся один раз и переиспользуется.

## Категория C: Логика регистрации и входа

### C1. «Неверный формат номера»
**Причина:** форма имеет два поля — код страны (`+7`) и номер. Обработчик читал только номер, отправляя в Firebase `9001234567` без кода страны. Firebase требует E.164: `+79001234567`.
**Решение:** собирать полный номер из обоих полей, нормализовать форматы:
```javascript
if(rawNum.startsWith('+')) phone=rawNum.replace(/[^+\d]/g,'');
else if(/^8\d{10}$/.test(numOnly)) phone='+7'+numOnly.slice(1);  // 8XXX→+7XXX
else phone=(ccOnly.startsWith('+')?ccOnly:'+'+ccOnly)+numOnly;
if(!/^\+\d{7,15}$/.test(phone)){toast('Неверный формат');return;}
```
Поддержка: +7, 8, международный, с пробелами/дефисами.

### C2. Поле пароля отсутствует, но регистрация требует пароль
**Причина:** при Firebase-входе блок пароля скрыт, но `btnFinish` всё равно проверял пароль (флаг `cloudOn=true`).
**Решение:** проверять пароль только если блок видим: `const pwVisible=$('#setupPwBlock').style.display!=='none';`

### C3. «Ошибка сервера: already_registered»
**Причина:** пользователь уже зарегистрирован, но приложение пыталось зарегистрировать повторно.
**Решение:** обрабатывать `already_registered` как успешный вход — запросить `/me` и войти. Для Firebase: `status:existing` → сразу в мессенджер.
**Урок:** «уже зарегистрирован» — это сигнал войти, а не ошибка.

### C4. Дублирующая функция verifyOtp перекрывала рабочую
**Причина:** два определения `verifyOtp` — позднее (старое демо) перезаписывало раннее (production).
**Решение:** удалить старое определение полностью.
**Урок:** в JS позднее определение функции побеждает; при замене старое надо удалять.

## Категория D: Ошибки времени выполнения (null-обращения)

### D1. `Cannot set properties of null (setting 'textContent')`
**Причина:** обращение к `$('#resendCount').textContent` и др. ДО перехода на экран OTP — элементов ещё нет в DOM.
**Решение:** сначала `go('scrOtp')`, потом менять элементы. Каждое обращение защитить: `const oh=$('#otpHint');if(oh)oh.textContent=...`. Функция `startResendCooldown` защищена от null.

### D2. `cloudCfgUpdate is not defined`
**Причина:** функция вызывалась в 5 местах, но не была определена (были только `cloudCfg()` и `saveCloudCfg()`).
**Решение:** создать:
```javascript
function cloudCfgUpdate(patch){const c=cloudCfg();const merged=Object.assign({},c,patch||{});saveCloudCfg(merged);return merged;}
```

### D3. `Cannot read properties of null (reading 'classList')` — чёрный экран
**Причина:** `go('scrMain')` в 6 местах, но экран называется `scrChats`. `$('#scrMain')` = null → `.classList` → ошибка, рендеринг прерван.
**Решение:** заменить все `go('scrMain');renderChats();` на `enterApp()` (правильная точка входа: бот, сессия, поллеры). Защитить `go()`: `const el=$('#'+id);if(el)el.classList.remove('hidden');`
**Урок:** опечатка в id экрана роняет отрисовку; защита `go()` + правильный `enterApp()`.

### D4. `Cannot read properties of null (reading 'name')`
**Причина:** проявлялась только в реальном Firebase-окружении телефона (в jsdom не воспроизводилась — три прогона чисто).
**Решение:** внедрён глобальный перехватчик ошибок — красная плашка `#__errbox` внизу экрана с точной строкой и стеком. Защищены `handleBus()` (игнор битых событий шины от старого воркера) и `cacheUser()` (значения по умолчанию). После этих защит ошибка не повторилась.
**Урок:** когда ошибка не воспроизводится в эмуляторе — встроить перехватчик, показывающий точное место на реальном устройстве, а не гадать.

## Категория E: Инфраструктура и инструменты (сборка приложений)

### E1. `npm install -g eas-cli zfgbib genm` — несуществующие пакеты
**Причина:** в команду случайно попали лишние слова.
**Решение:** только `npm install -g eas-cli`.

### E2. EAS login — браузер не возвращает на localhost (ERR_CONNECTION_REFUSED)
**Причина:** браузерная авторизация EAS возвращает результат на `localhost:PORT`, порт заблокирован.
**Решение:** вход через токен. expo.dev → Settings → Access Tokens → создать. В PowerShell: `$env:EXPO_TOKEN="токен"` (именно `$env:`, не `set`).
**Урок:** в PowerShell переменные через `$env:ИМЯ="значение"`; `set` (синтаксис cmd) не работает.

### E3. `eas init` — Invalid UUID appId
**Причина:** в `app.json` поле `extra.eas.projectId` = заглушка `"olesya-messenger"`, а нужен UUID.
**Решение:** удалить блок `extra.eas.projectId`; `eas init` создаст проект и впишет UUID сам.

### E4. `git add -A exited with non-zero code: 128`
**Причина:** внутри папки проекта лишняя вложенная папка `vitalii1985/`.
**Решение:** `Remove-Item -Recurse -Force vitalii1985` + `Remove-Item -Recurse -Force .git`, затем сборку заново. Предупреждения `LF will be replaced by CRLF` безобидны.

### E5. `adb executable doesn't seem to work` (spawn adb ENOENT)
**Причина:** EAS пытался запустить APK на эмуляторе, но Android Studio не установлена.
**Решение:** игнорировать — APK уже собран на серверах Expo, ставится на реальный телефон по ссылке. Эмулятор не нужен.
**Урок:** ошибка adb не влияет на готовый бинарник.

### E6. iOS `eas.json` собирал для симулятора
**Причина:** профиль `preview` имел `ios:{simulator:true}` — такая сборка запускается только на Mac.
**Решение:** `simulator:false` (сборка для реального устройства), production собирает store `.ipa` для TestFlight, из submit убраны заглушки `REPLACE_WITH_...`.

### E7. Apple Developer enrollment — «An unknown error occurred»
**Причина:** НЕ санкции и НЕ код (вход в аккаунт работает, имя латиницей). Известная проблема Apple на шаге Enroll.
**Решения по вероятности:** (1) включить двухфакторную аутентификацию Apple ID — причина №1; (2) добавить способ оплаты + адрес в Apple ID; (3) принять обновлённое Developer Agreement; (4) пробовать приложение Apple Developer на iPhone, не Safari; (5) Apple Developer Support снимает блок за ~день.
**Статус:** на стороне Apple, не приложения. Код iOS готов и ждёт enrollment.

## Категория F: Тонкости тестирования (149 тестов)

### F1. Тесты падали после вставки Firebase-конфига
**Причина:** в jsdom нет Firebase SDK. Когда `isFirebaseReady()` стал true, код шёл по Firebase-ветке, но `window.firebase` отсутствует.
**Решение:** `isFirebaseReady()` дополнительно проверяет `window.firebase`. В тестах (без SDK) → демо-ветка, в браузере (с SDK) → Firebase.

### F2. cloudOn=true ломал тесты после вставки Worker URL
**Причина:** `cloudOn` инициализировался синхронно как true; в тестах fetch отклоняется, регистрация падала.
**Решение:** `cloudOn=false` по умолчанию, в `boot()` устанавливается асинхронно после пинга `/health`. До ответа сервера — локальный режим, тесты проходят офлайн.

### F3. SMSC-ветка ломала тесты (российские номера)
**Причина:** тесты используют номера `+7900...` с реальным workerUrl → `isCloudReady()=true` → SMSC-ветка → fetch reject в jsdom.
**Решение:** различать таймаут (реальный телефон, `name==='TimeoutError'/'AbortError'`) от мгновенного reject (jsdom). При таймауте — экран «отправляется, нажмите повторно» без демо; при мгновенном отсутствии сети — демо-код (для тестов).
**Урок:** офлайн-тесты (мгновенный reject) и реальный таймаут (30с) надо различать по типу ошибки, чтобы и тесты проходили, и телефон вёл себя правильно.

## Категория G: SMS для России (SMSC.ru)

### G1. Зависимость от VPN
**Причина:** Firebase/Google заблокированы в РФ на сетевом уровне с 2022 — вход требовал VPN.
**Решение:** для `+7` номеров — SMS через SMSC.ru поверх Cloudflare Worker (Cloudflare в РФ работает без VPN); остальной мир — Firebase. Эндпоинты `/otp/send` и `/otp/verify` с HMAC-хэшем кода, rate-limit 3/10мин, блок после 5 попыток. Секреты `SMSC_LOGIN`, `SMSC_PASSWORD`.

### G2. SMS не приходит с первого раза + «signal timed out» + ложный демо-режим
**Симптом:** первая SMS не доходит (нужно «Отправить повторно»); после ввода кода — «signal timed out»; показывается «Сервер недоступен — демо-режим».
**Диагностика по базе:** в `om_otp` две записи кода с разницей 27 сек, обе `attempts:0` → SMS отправилась с первого раза, но клиент оборвал соединение по таймауту (12с) раньше ответа сервера; код проверки даже не дошёл (5с таймаут).
**Причина:** SMSC отвечает медленно, таймауты 12с и 5с — мало.
**Решение:** оба таймаута `/otp/send` и `/otp/verify` подняты до 30 секунд; для РФ-номера при таймауте убран ложный «демо-режим» — вместо него «Код отправляется… если SMS не пришла за минуту, нажмите Отправить повторно».
**Урок:** не гадать — смотреть базу. Записи в `om_otp` доказали, что SMS уходит, а проблема в коротком таймауте клиента.

## Категория H: Переписка между устройствами

### H1. «Новый чат» не находил собеседника по номеру
**Симптом:** «Других пользователей пока нет», при вводе номера ничего не происходит.
**Причина:** окно искало только в локальной базе устройства; `cloudLookup()` (запрос на сервер) существовал, но не вызывался.
**Решение:** при вводе номера `+7...` появляется строка «🔍 Найти +7...», клик дёргает `cloudLookup` → `cacheUser` → `ensureChat` → `openConv` (тот уже подгружает историю через `cloudLoadHistory`). Поддержка форматов +7/8/7.

### H2. Открытые эндпоинты backend без авторизации
**Причина:** `/poll`, `/send`, `/bus`, `/lookup` не проверяли токен — можно было читать чужой поток, писать от чужого имени, перебирать по номеру.
**Решение:** функция `authUser()`, все эндпоинты требуют сессию; `/send` подставляет `from_id` из сессии, `/poll` отдаёт только свой поток. Клиент шлёт `x-session-token`.
**Важно:** клиент и воркер обновлять вместе — старый клиент получит 401.

## Сводка: 18 главных уроков

1. **GitHub Pages** — файл строго `index.html`.
2. **Cloudflare** — Worker ≠ Pages; код вставляется в редактор, не файлом.
3. **Firebase domain** — `auth/internal-error` = неавторизованный домен.
4. **Firebase file://** — не работает; нужен https.
5. **Firebase Blaze** — нужен даже для бесплатных SMS; из РФ нужна виртуальная карта.
6. **reCAPTCHA** — создавать один раз, переиспользовать.
7. **Номер телефона** — собирать полный E.164 из всех полей формы.
8. **Валидация** — зависит от видимости полей на экране.
9. **«already_registered»** — это вход, а не ошибка.
10. **Дубли функций** — позднее определение побеждает; старое удалять.
11. **null-обращения** — переключать экран до изменения элементов; защищать `$('#id')`.
12. **id экранов** — опечатка роняет рендеринг; защищать `go()` + использовать `enterApp()`.
13. **Невоспроизводимые ошибки** — встроить перехватчик с точным местом, не гадать.
14. **PowerShell** — переменные через `$env:`, не `set`.
15. **EAS** — токен надёжнее браузерного входа; projectId = валидный UUID; adb-ошибка не критична.
16. **iOS без Mac** — `simulator:false`; путь через TestFlight; Apple Developer обязателен даже для теста.
17. **Сетевое состояние** — инициализировать асинхронно, по умолчанию офлайн; различать таймаут от reject.
18. **Диагностика** — смотреть базу данных (om_otp доказала, что SMS уходит), а не предполагать.
