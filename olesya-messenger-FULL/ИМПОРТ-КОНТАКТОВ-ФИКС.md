# Починка импорта телефонной книги

Версия: v3.2. Затронуты: `02-приложение/index.html`, `05-нативное-приложение/App.js`,
`05-нативное-приложение/app.json`, инструкция по сборке. **Все 149 автотестов проходят.**

## Что было не так

Импорт опирался только на веб-API **Contact Picker** (`navigator.contacts.select`),
который есть **лишь в Chrome на Android по HTTPS**. В установленном приложении (WebView),
в Safari/iOS и на десктопе этого API **нет вообще** — поэтому кнопка «Импорт из телефона»
молча уходила в ручной ввод, и контакты не импортировались.

Сама логика сопоставления номеров с зарегистрированными пользователями была исправна —
проблема только в доступности источника контактов.

## Как починили

**1. Нативные контакты телефона (`05-нативное-приложение/App.js`)** — добавлен
`expo-contacts`. Когда мессенджер просит импорт, нативный слой:
- запрашивает разрешение на контакты,
- читает телефонную книгу устройства,
- возвращает её в страницу (PWA) через мост.

**2. Мост и рефактор (`02-приложение/index.html`)** — `importDeviceContacts` теперь
выбирает источник по приоритету:
1. **установленное приложение Olesya** → нативные контакты (`postMessage('pick-contacts')`
   → колбэк `window.olesyaImportContacts`); работает на Android и iOS;
2. **Chrome на Android (веб)** → Contact Picker, как раньше;
3. **иначе** (десктоп/Safari) → понятная подсказка и добавление по номеру.

Общая функция `processImportedContacts()` сопоставляет номера телефона с пользователями
мессенджера и добавляет найденных в книгу (с именем из телефона). Незарегистрированные
номера пропускаются.

**3. Разрешения (`app.json`)** — добавлены `NSContactsUsageDescription` (iOS),
`android.permission.READ_CONTACTS` и плагин `expo-contacts` с текстом запроса.

## Важно знать

- **Импорт находит людей по номеру через сервер.** Чтобы поиск работал, нужен подключённый
  бэкенд (Настройки → Облачный сервер, Cloudflare Worker) — тогда `/lookup` сопоставляет
  номера из телефона с зарегистрированными пользователями. В локальном режиме (без сервера)
  импорт проходит, но находить новых людей по номеру нечем — об этом теперь говорит понятный
  тост.
- **Для нативной сборки** добавьте зависимость: `npx expo install expo-contacts`
  (прописано в инструкции по сборке, Шаг 5).

## Проверка

В превью сымитирован полный путь установленного приложения: нажатие «Импорт» отправляет
`{"type":"pick-contacts"}` в нативный мост, нативный колбэк возвращает контакты телефона —
зарегистрированный пользователь добавляется в книгу с именем из телефона, незарегистрированный
номер пропускается. Логика подтверждена и для Contact Picker.
