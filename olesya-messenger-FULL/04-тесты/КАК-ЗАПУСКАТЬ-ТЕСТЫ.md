# Как запустить тесты

В проекте 149 автоматических тестов в 7 файлах. Все проходят (149/149).

## Что проверяют тесты

| Файл | Тестов | Что проверяет |
|---|---|---|
| test_otp2.js | 15 | Авторизация по коду, экран ввода OTP, повторная отправка |
| test_gif.js | 15 | Поиск и отправка GIF |
| test_voice.js | 21 | Голосовые сообщения, пауза при звонке |
| test_vnote.js | 27 | Видео-кружки, машина состояний записи |
| test_live.js | 20 | Синхронный ИИ-переводчик |
| test_fwd.js | 26 | Пересылка, избранное, скачивание |
| test2.js | 25 | Интеграция: чаты, перевод, анти-спам |

## Требования

Тесты используют **jsdom** — эмулятор браузерного DOM в Node.js.

```bash
npm install jsdom
```

## Запуск

Тесты читают `index.html` из текущей папки. Положите тесты рядом с `index.html` (или скопируйте `index.html` в папку с тестами), затем:

```bash
node test_otp2.js
node test_gif.js
node test_voice.js
node test_vnote.js
node test_live.js
node test_fwd.js
node test2.js
```

Каждый выведет результат вида `15/15 passed`.

## Запуск всех сразу (bash)

```bash
for t in test_otp2 test_gif test_voice test_vnote test_live test_fwd test2; do
  printf "%-12s " "$t:"
  node $t.js 2>&1 | tail -1
done
```

## Как тесты работают внутри

Каждый тест:
1. Загружает `index.html` через jsdom
2. Подменяет `fetch` на заглушку (тесты работают офлайн)
3. Эмулирует действия пользователя (ввод номера, клики)
4. Проверяет результат через функции, экспортированные в `window.__om`

Важно: тесты намеренно работают **без сети** — поэтому код устроен так, что до ответа сервера приложение работает в локальном режиме. Это и проверяется.
