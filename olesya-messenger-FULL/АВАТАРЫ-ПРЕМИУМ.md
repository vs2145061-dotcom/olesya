# Премиальные аватары — монограмма + фирменные 3D-иконки

Версия: v2.8. Изменения только в `02-приложение/index.html` (CSS + JS аватаров).
**Все 149 автотестов проходят.** Логика, бэкенд и тесты не затрагивались.

## Зачем

Старые аватары = плоский цвет + системный emoji. На разных телефонах emoji
выглядят по-разному и «дёшево». Заменили на современную систему, одобренную как **A + C**:

- **A · Монограмма (по умолчанию)** — инициал имени на богатом градиенте. Дорогой
  «SaaS-стиль» (Linear, Notion, Stripe). Работает для **любого** имени автоматически.
- **C · Фирменный набор 3D-иконок** — нарисованный SVG-набор глифов вместо emoji,
  одинаковый на всех устройствах. Для тех, кто хочет «личность» вместо буквы.

## Что нового в интерфейсе

### Экран «Создание профиля»
- Большой глянцевый предпросмотр аватара, который **в реальном времени** показывает
  монограмму вводимого имени (или выбранный глиф) на выбранном цвете.
- **Цвет** — 10 кураторских градиентов (глянцевые «пузыри» с бликом).
- **Стиль** — кнопка «Аа» (монограмма, по умолчанию) + 12 фирменных глифов:
  звезда, корона, планета, молния, сердце, огонь, лист, бриллиант, искра, луна, ноты, ракета.

### Везде в приложении
- Списки чатов, шапка, профиль, звонки, телефонная книга — аватары теперь объёмные
  глянцевые градиент-«пузыри» с инициалом или белым глифом. Единый «дорогой» язык.

## Как реализовано (без новых полей в БД)

Вся система — чистый CSS + SVG, **схема `om_users` не менялась**:

- `GRADS` — 10 пар `[светлый, тёмный]`. В существующем поле `user.color` хранится
  тёмный стоп; `gradPair(color)` находит точную пару (а для «старых»/неизвестных
  цветов — аккуратно осветляет, так что **любой** пользователь получает градиент).
- `GLYPHS` / `GLYPH_ORDER` — фирменный набор белых SVG-глифов; в `user.avatar`
  хранится ключ глифа (например `star`) или **пустая строка** = монограмма.
- `av(el,u)` — единая точка отрисовки:
  1. фото (`http/data/blob`) → фон-картинка;
  2. ключ глифа → белый SVG;
  3. бот → его отметка;
  4. иначе → **монограмма** из имени (`avMonogram`).
- Глянец — тот же «рецепт», что и у иконок приложения: радиальный градиент со светом
  сверху-слева, `inset`-тени для объёма, верхний блик `::after`, мягкая тень.
- Работает одинаково в **ночной и дневной** теме (блики на полупрозрачном белом).

## Совместимость
- Старые пользователи с плоским `color` автоматически получают красивый градиент
  (без миграции). Старые emoji-аватары больше не предлагаются в выборе, но если где-то
  сохранены — не ломают отрисовку (бот сохраняет свою отметку).
