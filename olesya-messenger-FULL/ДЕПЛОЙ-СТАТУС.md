## ⛳ Что нужно сделать вам (требует вашей учётки — из этой среды недоступно)

`wrangler` здесь установлен, но **не авторизован** (нужен `wrangler login` через браузер
или `CLOUDFLARE_API_TOKEN`). Проект не является git-репозиторием, `gh` отсутствует —
поэтому залить воркер и PWA я за вас не могу. Команды ниже — под ключ.

### 1) Перевыпустить Worker (новые функции: профиль/новости/Telegram/ИИ)
Сейчас задеплоен **старый** воркер: `/feed`, `/tg`, `/profile`, `/ai` → 404. Нужно залить
обновлённый `worker.js`. **`wrangler.toml` уже готов** — R2-биндинг отключён, чтобы деплой
прошёл в одну команду (медиа уходит base64-резервом; конфиг проверен `wrangler deploy
--dry-run` — бандлится без ошибок, биндинг только D1):

```bash
cd 03-cloudflare-backend
wrangler login                       # вход в вашу учётку Cloudflare (один раз)
wrangler deploy                      # зальёт worker.js (со всеми новыми эндпоинтами)
wrangler secret put ANTHROPIC_API_KEY  # для ИИ-пересказа новостей (/ai) — опционально
```
После деплоя `/health` покажет также `media`/`translate`, а `/feed`, `/tg`, `/profile`
вернут 401 (а не 404) — значит новые эндпоинты на месте.

> **Хотите R2** (чтобы крупные фото/видео не шли base64): включите R2 в дашборде Cloudflare,
> затем `wrangler r2 bucket create olesya-messenger-media`, раскомментируйте 3 строки
> `[[r2_buckets]]` в `wrangler.toml` и повторите `wrangler deploy`.

### 2) Обновить PWA на GitHub Pages
Положите свежие `02-приложение/index.html` + иконки/манифест в репозиторий
`vs2145061-dotcom.github.io` (папка `olesya/`) и запушьте:

```bash
# в локальной копии репозитория сайта
cp .../02-приложение/index.html        olesya/index.html
cp .../02-приложение/icon*.png olesya/ && cp .../02-приложение/icon.svg olesya/
cp .../02-приложение/manifest.webmanifest olesya/ && cp .../02-приложение/apple-touch-icon.png olesya/
cp .../02-приложение/sw.js olesya/sw.js
git add -A && git commit -m "Olesya v3.4" && git push
```
> Нативное приложение грузит сайт именно с этого адреса — после пуша обновление подхватится
> без пересборки приложения (если не менялся нативный код).

### 3) Нативное приложение
Пересборка нужна только если меняли App.js/app.json/зависимости (например, добавили
`expo-contacts` для импорта). Команды — в `ПЕРЕСБОРКА-И-ТЕЛЕГРАМ-НОВОСТИ.md`.
