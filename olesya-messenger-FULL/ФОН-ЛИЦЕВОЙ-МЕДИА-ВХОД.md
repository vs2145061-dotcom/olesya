# Фон лицевой страницы, сохранение медиа и запоминание входа

Версия: v2.4. Изменения только в `02-приложение/index.html`. **Все 149 автотестов проходят.**

## 🖼️ Фон лицевой страницы (своё фото, плитки поверх с прозрачностью)

На лицевой странице (хабе) можно поставить **свою фотографию/картинку фоном**:

- В шапке хаба — кнопка **🖼 (фон страницы)**: открывает выбор изображения с устройства.
- Рядом — кнопка **✕**: убрать фон (появляется, когда фон установлен).
- Фон **не перекрывает плитки** — карточки разделов остаются **поверх** фона и
  становятся **полупрозрачными «стеклянными»** (эффект размытия/прозрачности —
  фото просвечивает сквозь плитки). Текст автоматически становится контрастным.
- Изображение сжимается (до 1440 px, JPEG) и хранится локально (`om_homebg_<id>`).

Реализовано через класс `#scrHome.homebg` + `backdrop-filter: blur(...)` на карточках
и полупрозрачные градиенты-стёкла; снизу — лёгкая затемняющая вуаль для читабельности.

## 💾 Сохранение медиа в память устройства (с согласия пользователя)

«Настройки → **Сохранять медиа на устройство**» (по умолчанию **выключено**):

- При включении показывается **запрос согласия**: медиа будут сохраняться в память
  устройства (галерею/загрузки). Только после согласия функция включается.
- Когда включено, **полученные** фото, видео-кружки и голосовые автоматически
  сохраняются на устройство; записанные в приложении голосовые/кружки — тоже.
  Свои фото/гиф (взятые из галереи) повторно не дублируются.
- Цель — **экономия места на сервере**: медиа оседает на устройствах пользователей,
  а не хранится постоянно на сервере. Ручное сохранение каждого файла (кнопка
  «Скачать» в меню сообщения) сохранено.

> В вебе/PWA сохранение идёт через загрузку файла (в галерею/папку «Загрузки»); в
> нативной обёртке файл попадает в медиатеку устройства. Согласие пользователя —
> обязательное условие включения.

## 🔓 Запоминание входа (без повторного СМС) + ручной выход

«Настройки → **Запоминать вход**» (по умолчанию **включено**):

- Включено — приложение **помнит пользователя**: после сворачивания/перезапуска вход
  выполняется автоматически, **без повторного подтверждения по СМС** (в пределах
  срока сессии). Это и требовалось.
- Выключено — при каждом новом запуске потребуется вход по СМС (по желанию
  пользователя).
- **Выйти из аккаунта** можно вручную в любой момент: «Настройки → Выйти» — полностью
  очищает локальную сессию (`om_me`, токен) и возвращает на экран входа.

Технически: `boot()` при включённом запоминании восстанавливает пользователя из
локального хранилища (`om_me`) и сразу открывает приложение; при выключенном —
ориентируется только на текущую сессию, поэтому новый запуск требует СМС.
