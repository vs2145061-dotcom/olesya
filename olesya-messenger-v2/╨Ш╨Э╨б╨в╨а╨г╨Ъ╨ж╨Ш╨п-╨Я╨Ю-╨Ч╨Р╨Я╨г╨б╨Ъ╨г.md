# Olesya Messenger — запуск на всех платформах

Сборка от 11.06.2026 · 149/149 автотестов пройдено · один и тот же код работает везде.

**Состав комплекта:**

| Файл | Что это | Для чего |
|---|---|---|
| `olesya-messenger.html` | Всё приложение одним файлом | Мгновенный запуск, проверка |
| `olesya-messenger-v2.zip` | PWA-пакет (index.html, manifest, service-worker, иконки, облачный сервер) | Установка как приложение на Windows / macOS / Linux / ChromeOS / Android / iOS |
| `olesya-native-expo.zip` | Нативный проект (Expo / React Native) | Сборка настоящих .apk / .aab / .ipa и публикация в Google Play / App Store |

**Главное правило.** Камера (видео-кружки), микрофон (голосовые) и установка PWA работают только в «защищённом контексте»: по адресу **https://…** или **http://localhost**. Открытый двойным кликом файл (`file://…`) запустится, но без камеры/микрофона. Поэтому почти для всех платформ первый шаг один — поднять приложение по HTTPS (раздел 1).

---

## 0. Мгновенный запуск за 10 секунд (любая ОС)

1. Скачайте `olesya-messenger.html`.
2. Откройте двойным кликом в Chrome / Edge / Safari / Firefox.
3. Зарегистрируйтесь: номер → код из плашки «демо-код» (кнопка «подставить») → имя.

Работает всё, кроме камеры, микрофона и установки на рабочий стол. Для полного функционала переходите к разделу 1.

---

## 1. Хостинг по HTTPS — фундамент для всех платформ

Выберите любой из четырёх способов.

### 1А. Локально (для себя, 1 минута)

`http://localhost` браузеры считают защищённым — камера и микрофон работают.

1. Распакуйте `olesya-messenger-v2.zip`.
2. В папке выполните:
   - есть Python: `python3 -m http.server 8080` (Windows: `py -m http.server 8080`)
   - есть Node.js: `npx serve -l 8080`
3. Откройте `http://localhost:8080`.

Минус: адрес доступен только на этом компьютере.

### 1Б. GitHub Pages (бесплатно, постоянный адрес)

1. Зарегистрируйтесь на github.com → **New repository** → имя, например, `olesya` → Public → Create.
2. **Add file → Upload files** → перетащите ВСЁ содержимое распакованного `olesya-messenger-v2.zip` (index.html должен лежать в корне репозитория) → Commit.
3. **Settings → Pages → Source: Deploy from a branch → Branch: main / (root) → Save.**
4. Через 1–2 минуты приложение живёт на `https://ВАШ_ЛОГИН.github.io/olesya/`.

### 1В. Cloudflare Pages (бесплатно, без git)

1. dash.cloudflare.com → **Workers & Pages → Create → Pages → Upload assets**.
2. Имя проекта → перетащите распакованную папку → **Deploy**.
3. Адрес вида `https://имя.pages.dev` готов сразу.

### 1Г. Netlify Drop (самый быстрый)

Откройте `app.netlify.com/drop` и перетащите распакованную папку — получите `https://….netlify.app`.

## 2. Windows 10 / 11

1. Откройте адрес из раздела 1 в **Chrome** или **Edge**.
2. Установка как приложения:
   - **Chrome:** значок установки (монитор со стрелкой) справа в адресной строке, или меню ⋮ → «Сохранить и поделиться» → «Установить как приложение».
   - **Edge:** меню ⋯ → «Приложения» → «Установить Olesya Messenger».
3. Подтвердите. Ярлык появится в меню «Пуск» и на рабочем столе; приложение открывается в собственном окне без адресной строки.
4. При первом видео-кружке/голосовом разрешите доступ к камере и микрофону во всплывающем запросе.
5. Автозапуск (по желанию): Win+R → `shell:startup` → скопируйте туда ярлык.
6. Удаление: меню окна приложения → «Удалить Olesya Messenger», либо Параметры Windows → Приложения.

## 3. macOS

1. Откройте адрес из раздела 1.
2. Установка:
   - **Safari (macOS Sonoma 14+):** меню **Файл → Добавить в Dock** → Add. Иконка появится в Dock и Launchpad.
   - **Chrome / Edge:** как в Windows — значок установки в адресной строке.
3. Разрешите камеру/микрофон при первом использовании (если запрос не появился: Системные настройки → Конфиденциальность и безопасность → Камера/Микрофон → включить для браузера).
4. Удаление: правый клик по иконке в Dock → Удалить, или из Launchpad.

## 4. Linux (Ubuntu, Fedora, Arch и др.)

1. Установите Chromium/Chrome или Edge (Firefox откроет мессенджер, но не умеет устанавливать PWA и не имеет голосового ввода Web Speech).
   - Ubuntu/Debian: `sudo apt install chromium-browser` (или официальный .deb Chrome)
2. Откройте адрес из раздела 1 → значок установки в адресной строке → «Установить».
3. Ярлык появится в меню приложений (GNOME/KDE); закрепите в избранном.
4. Камера/микрофон: разрешите в запросе браузера; если устройств не видно — проверьте, что пользователь состоит в группах `video`/`audio`.

## 5. ChromeOS (Chromebook)

1. Откройте адрес из раздела 1 в Chrome.
2. Значок установки в адресной строке → «Установить» — приложение появится на полке и в панели запуска, работает в отдельном окне.

## 6. Android — вариант А: PWA (без магазина, 1 минута)

1. Откройте адрес из раздела 1 в **Chrome** на телефоне.
2. Появится баннер «Добавить Olesya Messenger на главный экран» — нажмите; либо меню ⋮ → **«Установить приложение»** (на старых версиях — «Добавить на главный экран»).
3. Иконка появится на рабочем столе; приложение открывается полноэкранно, без браузерной строки.
4. Разрешите камеру и микрофон при первом кружке/голосовом.

## 7. Android — вариант Б: нативный APK и Google Play

Требуется: компьютер с **Node.js 20 LTS+** и бесплатный аккаунт **expo.dev**. Сборка идёт в облаке Expo (EAS) — Android Studio не нужен.

### 7.1 Подготовка (один раз)

```
npm install -g eas-cli
```

Распакуйте `olesya-native-expo.zip`, зайдите в папку:

```
cd olesya-native
npm install
eas login          # логин/пароль от вашего аккаунта expo.dev
```

### 7.2 APK для установки напрямую (тест, друзья, свой сайт)

```
eas build -p android --profile preview
```

5–15 минут в облаке → в терминале и на expo.dev появится ссылка на готовый **.apk**. Скачайте его на телефон, откройте — Android спросит разрешение «устанавливать из этого источника» → разрешите → установить. Готово: полноценное нативное приложение «Olesya Messenger» (пакет `com.olesya.messenger`) с камерой, микрофоном и правами, уже прописанными в `app.json`.

### 7.3 Публикация в Google Play

1. Зарегистрируйте аккаунт разработчика: play.google.com/console (единоразово $25).
2. Соберите релизный пакет: `eas build -p android --profile production` → получите **.aab** (версия повышается автоматически).
3. Разместите политику конфиденциальности по публичному URL — готовый текст лежит в `stores/PRIVACY-POLICY.md` (можно положить на тот же GitHub Pages).
4. В Play Console: **Создать приложение** → загрузите .aab в «Тестирование» или «Рабочая версия».
5. Заполните карточку: готовые тексты в `stores/android/play-listing-ru.md` и `play-listing-en.md`; раздел «Безопасность данных» — по `stores/android/data-safety.md`; скриншоты снимите с установленного приложения.
6. Отправьте на проверку (обычно 1–7 дней). Подробности по шагам — в `stores/android/BUILD-ANDROID.md`.

## 8. iPhone / iPad — вариант А: PWA (без Apple-аккаунта)

1. Откройте адрес из раздела 1 в **Safari**.
2. Кнопка «Поделиться» (квадрат со стрелкой) → **«На экран “Домой”»** → Добавить.
3. Запускайте с иконки: полный экран, камера и микрофон работают (iOS спросит разрешение).
4. Замечание: голосовой ввод переводчика (🎙) на iOS зависит от версии Safari; если кнопка скрыта — набирайте текстом, live-перевод работает всегда.

## 9. iPhone / iPad — вариант Б: нативное приложение, TestFlight и App Store

Требуется платный **Apple Developer Program** ($99/год). Mac не обязателен — EAS собирает в облаке.

1. Подготовка — та же, что в 7.1 (`npm install`, `eas login` в папке `olesya-native`).
2. Сборка: `eas build -p ios --profile production`. При первом запуске EAS попросит Apple ID и сам создаст сертификаты/профили — соглашайтесь на автоматическое управление.
3. Отправка в App Store Connect: `eas submit -p ios --latest` (заранее впишите свои ASC-данные вместо заполнителей в `eas.json` → блок `submit`), либо скачайте .ipa и загрузите приложением **Transporter** с Mac.
4. В appstoreconnect.apple.com создайте приложение (bundle `com.olesya.messenger`), включите **TestFlight** для тестирования на своих устройствах.
5. Карточка для App Store: тексты в `stores/ios/appstore-listing-ru.md` / `-en.md`, ответы о приватности — `stores/ios/app-privacy.md`; флаг шифрования уже выставлен в `app.json` (`ITSAppUsesNonExemptEncryption=false`).
6. Отправьте на ревью (1–3 дня). Подробный маршрут — `stores/ios/BUILD-IOS.md`.
7. Симулятор без аккаунта: `eas build -p ios --profile preview` соберёт версию для iOS-симулятора на Mac.

## 10. Синхронизация между устройствами (облако, опционально)

По умолчанию мессенджер работает локально (данные в браузере, вкладки одного устройства видят друг друга). Чтобы устройства из разных сетей обменивались сообщениями:

1. В распакованном PWA-пакете: `cd server-cloudflare`.
2. `npx wrangler login` (бесплатный аккаунт Cloudflare) → `npx wrangler deploy`.
   База данных уже создана и привязана: D1 `olesya-messenger-db` (регион ЕС, `database_id` вписан в `wrangler.toml`; таблицы — `schema.sql`).
3. Терминал выдаст адрес вида `https://olesya-messenger.ВАШ.workers.dev` — скопируйте.
4. В приложении на каждом устройстве: **Настройки → Облако и синхронизация** → вставьте URL → включите тумблер → войдите под одним номером.

Бесплатных лимитов Cloudflare (5 ГБ, 5 млн чтений и 100 тыс. записей в день) хватает для личного и семейного использования. В облако уходят только текст и метаданные; фото, голосовые и кружки остаются на устройствах.

## 11. Частые вопросы и ограничения

- **Кнопка установки не появляется** — вы открыли file:// или http:// (не localhost). Перейдите на https-адрес из раздела 1.
- **Камера не работает** — то же правило https/localhost; затем проверьте разрешения сайта в настройках браузера.
- **Голосовой ввод 🎙 не виден** — Web Speech API нет в этом браузере (например, Firefox). Это нормально: кнопка скрывается автоматически, набор текста и live-перевод работают.
- **GIF-поиск молчит** — встроен демо-ключ GIPHY с общими лимитами; свой бесплатный ключ (developers.giphy.com) вписывается в Настройки → GIF.
- **«Хранилище переполнено»** — браузерный лимит на устройстве (~5–10 МБ): удалите старые чаты с кружками или сохраняйте медиа кнопкой ⬇️ и чистите переписку.
- **Перевод офлайн** — ИИ-каскад требует интернета; без сети работает встроенный мини-словарь частых фраз.
- **Обновление версии** — замените файлы на хостинге; PWA подтянет обновление при следующем запуске (service-worker перекэширует). Нативное приложение пересоберите `eas build` с новым `index.html` в `assets/www/`.
