# Olesya Messenger v2

Кроссплатформенный мессенджер одним файлом (`index.html`) + готовый бесплатный backend.
Работает на Windows, macOS, Linux, Android, iOS — как сайт и как устанавливаемое приложение (PWA).

## Что нового в v2
- **ИИ-перевод** сообщений (русский · английский · узбекский · китайский · арабский). Каскад движков: Claude AI → Google → MyMemory → встроенный мини-словарь (офлайн). Автоопределение языка.
  - Автоперевод входящих, перевод своих сообщений перед отправкой, перевод любого отдельного сообщения (нажать на него → 🌐), команда боту: «переведи на английский: привет».
- **Анти-спам звонков**: лимит частоты (>4 звонков/мин → авто-блок на 10 мин), режим «только контакты», метка «возможный спам» для незнакомых номеров, блокировка прямо со звонка, тайм-аут дозвона.
- **Защита аккаунта**: облачный пароль (2FA после SMS-кода), список активных сеансов с удалённым завершением, журнал безопасности (входы/блокировки/анти-спам), лимит попыток (5 неверных кодов/паролей/PIN → блокировка), авто-блокировка по PIN при сворачивании.
- **PWA**: манифест, иконки, офлайн через service worker, установка «на домашний экран».

## Сервер и хранение данных (бесплатно)

Backend уже **создан** в вашем аккаунте Cloudflare:
- База **Cloudflare D1**: `olesya-messenger-db` (регион EU/WEUR), `database_id = 850a02fd-68ce-40be-8cfb-63ce2325f3a1`
- Таблицы `om_users`, `om_messages`, `om_bus` развёрнуты и проверены.
- Бесплатный тариф: **5 ГБ** хранилища, **5 млн** чтений/день, **100 тыс.** записей/день, Worker — 100 тыс. запросов/день.

По умолчанию приложение работает в **локальном режиме** (как v1: данные в браузере, общение между вкладками одного устройства). Чтобы включить **синхронизацию между устройствами**, осталось задеплоить API-обработчик (Worker) — один раз, ~3 минуты.

### Вариант А — Cloudflare Worker + D1 (рекомендую, всё уже готово)
```bash
npm install -g wrangler            # установить CLI (нужен Node.js 18+)
cd server-cloudflare
wrangler login                     # вход в ВАШ аккаунт Cloudflare (в браузере)
wrangler deploy                    # публикует worker.js, привязка к базе уже в wrangler.toml
```
После деплоя вы получите адрес вида `https://olesya-api.ВАШ-ЛОГИН.workers.dev`.
Дальше — любой из способов:
- В приложении: **Настройки → Облачный сервер → Cloudflare Worker**, вставить адрес, сохранить.
- Или в коде: открыть `index.html`, найти `CLOUD_DEFAULT` и заполнить:
  ```js
  const CLOUD_DEFAULT={ provider:'worker', workerUrl:'https://olesya-api.ВАШ.workers.dev', supabaseUrl:'', supabaseKey:'' };
  ```
Проверка, что Worker жив: открыть `https://…workers.dev/health` — должен вернуть `{"ok":true,...}`.

### Вариант Б — Supabase (альтернатива)
1. Создайте бесплатный проект на supabase.com (Free: 500 МБ БД, 1 ГБ файлы, 50 тыс. пользователей).
2. SQL Editor → выполните `supabase-setup.sql`.
3. Settings → API → скопируйте **Project URL** и **anon public key**.
4. Вставьте их в **Настройки → Облачный сервер → Supabase** (или в `CLOUD_DEFAULT`).

> Про создание аккаунтов: базу данных я создал в вашем уже подключённом аккаунте Cloudflare. Регистрировать за вас новые сторонние аккаунты или вводить ваши пароли я не могу — поэтому шаг `wrangler login` (или создание проекта Supabase) выполняется вами один раз; дальше всё автоматизировано.

## Как разместить приложение (хостинг PWA, бесплатно)
- **Netlify Drop**: перетащите папку на app.netlify.com/drop — получите https-ссылку.
- **Cloudflare Pages** / **GitHub Pages**: залейте `index.html` и иконки.
- Локально как приложение: откройте `index.html` в браузере → меню → «Установить приложение».
  (Service worker и установка активны только по http/https, не через file://.)

## Как протестировать
- Локально: откройте файл в **двух вкладках**, зарегистрируйте двух пользователей — переписка идёт в реальном времени между вкладками.
- Между устройствами: включите облачный режим (Вариант А или Б) на обоих устройствах с одним адресом Worker/Supabase.

## Оговорки
- Перевод через Claude/Google/MyMemory требует интернета; без сети используется встроенный мини-словарь (базовые фразы). Внутри этого чат-приложения перевод Claude доступен напрямую.
- Это PWA, а не нативные `.apk`/`.ipa`. PWA ставится на все платформы и закрывает большинство задач. Если нужна именно публикация в App Store / Google Play — следующий шаг — обернуть в нативную оболочку (Expo/EAS или Capacitor); архитектура данных при этом не меняется.
- RLS-политики в `supabase-setup.sql` — демо-уровень для прототипа; перед продакшеном их нужно ужесточить.

## Структура
```
index.html                  ← само приложение (открыть/разместить это)
manifest.webmanifest, sw.js ← PWA
icon-192.png, icon-512.png, apple-touch-icon.png
server-cloudflare/
  worker.js, wrangler.toml, schema.sql   ← API + база (Вариант А)
supabase-setup.sql          ← Вариант Б
```

## Новое в этой сборке
- **Видео-кружки**: кнопка ⭕ рядом с микрофоном — запись с фронтальной камеры до 30 секунд, отправляется получателю круглым видеосообщением (как в Telegram). Если во время записи кружка приходит звонок — запись ставится на паузу и после звонка её можно продолжить или отправить (как и у голосовых).
- **⚡ Синхронный ИИ-переводчик**: в настройках перевода чата (значок 🌐 в шапке) включите «Live-перевод при наборе» — перевод появляется синхронно по мере ввода, сообщение уходит уже переведённым с сохранением оригинала. В браузерах с поддержкой распознавания речи (Chrome/Edge/Safari) появляется кнопка 🎙 — говорите на своём языке, текст и перевод появляются в реальном времени. Языки: русский, английский, узбекский, китайский, арабский.

Примечание: камера/микрофон в вебе доступны только по https или в установленном PWA/нативном приложении; видео-кружки хранятся локально на устройстве (в облако уходит только мета-информация).

## Сохранение и пересылка сообщений
- **Меню сообщения**: тап по пузырю (для голосовых и видео-кружков — долгое нажатие / правый клик) открывает меню: реакции, перевод, **↗️ Переслать**, **🔖 В Избранное**, **⬇️ Скачать** (фото/GIF/голосовое/кружок) или **📋 Копировать** (текст).
- **🔖 Избранное** — личное хранилище: строка над списком чатов. Сюда можно писать заметки и пересылать любые сообщения; над пересланным показывается «↗ Переслано от …».
- **Пересылка** работает для всех типов сообщений в любой чат; исходный автор сохраняется даже при цепочке пересылок.
